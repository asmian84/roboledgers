5.  If Antigravity Gets Stuck

Problem: Can't find where to add updateTransactionContext
Paste this to Antigravity:
Search the codebase for localStorage.getItem and find where transactions are parsed back into an array. The line after that parse is where you add: window.RuleEngine.updateTransactionContext(transactions) — using whatever the array variable is named.

Problem: Import error — SignalFusionEngine not found
The new RuleEngine.js has this import at the top:
import SignalFusionEngine, { getConfidenceTier } from './SignalFusionEngine.js';
Paste this to Antigravity:
Check that SignalFusionEngine.js exists at src/services/SignalFusionEngine.js. If the project uses a different import convention (e.g. @/services/ prefix or absolute paths), update only the import path in RuleEngine.js to match — do not change anything else.

Problem: Antigravity tries to merge instead of replacing RuleEngine
Paste this to Antigravity:
Do not merge. Open src/services/RuleEngine.js, select all content, delete it, then paste the provided replacement file in full. The old waterfall logic is completely replaced by the fusion engine. Nothing from the old file needs to be preserved.

Problem: result.tier is undefined
.tier is added inside applyRules() in the new RuleEngine. If you're calling categorize() or another custom wrapper instead of applyRules() directly, either update your call site or add this line after any direct call:
result.tier = getConfidenceTier(result.confidence);
Import getConfidenceTier from SignalFusionEngine.js where needed.
